Key Concepts

  1. every participant has their own replicated copy of the ledger,share the ledger information,share the processes updating the ledger.
  2. what is hyperledger fabric? is a system by which participants manage their transactions.
  3. 2 features: has a ledger; uses smart contracts;
  4. different from other blockchain systems: it is private and permissioned.
  5. offers the ability to create channels.
  6. 通道用来保持事务在整个网络上的私有，在一个通道里，又使用私有数据集合来保证子组织数据的私有。

private data


通常一个通道上的账本是一致的，这时候，要想有私有数据，就要再创建一个通道，但这么做的开销很大（链码版本、通道策略等等），所以Fabric 1.2版本开始支持创建私有数据集，即一个通道上的组织子集，去背书、提交、查询，不需要创建单独的通道。

私有数据集包含2个元素：

  1. private data：
    存储在私有数据库（SideDB）中，发送给有权限查看的组织。不同于一般数据，私有数据在组织间通过gossip协议点对点传输，所以orderer服务是看不到私有数据的。
  2. hash of data：
    在通道上所有peer的账本中，用于事务的校验，作为交易的证据。


私有数据集 vs 一个单独的通道
要求账本对一组peers保密，使用通道。

包含私有数据集的事务流和普通的事务流不同，私有数据的交易流程：

  1. app提交一个读写私有数据的请求，到授权的背书结点。
  2. 背书结点模拟执行交易，将私有数据存储，并通过gossip协议发送到授权的peer。
  3. 背书结点把处理结果放在公共数据中返回给app，包括私有数据的键值和内容的hash，私有数据本身不会返回给应用。
  4. app提交交易到orderer结点，orderer结点照常打包block，然后分发到各个peer结点，这样，所有的peer都能独立的验证包含私有数据的交易，而不需要知道私有数据的内容。
  5. 区块验证阶段，被授权的peer判定自己是否有权读取私有数据。如果有权限，会先验证本地的私有数据来判断是否在背书阶段就收到了私有数据，如果没有，就会从其它peer拉取。
