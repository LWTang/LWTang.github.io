flask blueprint

目录结构：

  • app
    • __init__.py
    • main
      • __init__.py
      • views.py
      • errors.py
    • auth
      • __init__.py
      • views.py
      • errors.py

通过实例化一个Blueprint类来创建蓝本。这里import views和errors要放在实例化之后，因为views和errors模块要import main，如果放在实例化之前的话会导入出错。

在蓝本中编写错误处理程序稍有不同，如果使用errorhandler装饰器，那么只有蓝本中的错误才能触发处理程序。要想注册应用全局的错误处理程序，必须使用app_errorhandler装饰器。

蓝本的视图函数主要有两点不同：
第一，路由装饰器由蓝本提供，因此使用的是main.route，而非app.route；
第二，url_for()函数的用法不同，在应用的路由中，默认为视图函数的名称，例如，index()视图函数的url可使用url_for(‘index’)获取。在蓝本中，flask会为蓝本中的全部端点加上一个命名空间，这样就可以在不同的蓝本中使用相同的端点名定义视图函数。因此，上面代码中视图函数index()注册的端点名是main.index，其url使用url_for(‘main.index’)获取。

flask login_required

装饰顺序从上到下，执行顺序从下到上。secret()函数受login_required装饰器的保护，禁止未授权的用户访问，得到的函数又注册为一个Flask路由，如果调换两个装饰器的顺序，得到的结果是错误的。
由于login_required装饰器，如果未通过身份认证的用户访问这个路由，Flask-Login将拦截请求，把用户发往登录页面。
